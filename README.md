# Smart Commit\n\n**Smart Commit** is a highly customizable CLI utility for creating Git commits interactively. It provides a user-friendly interactive prompt to help you create consistent and well-formatted commit messages. The tool includes advanced features such as:\n\n- **Interactive Prompts:** Configure and optionally disable prompts for commit type, scope, summary, body, footer, ticket, and CI tests.\n- **Template-Based Commit Message:** Customize your commit message template using placeholders:\n  - `{type}`: Commit type (e.g., feat, fix, etc.)\n  - `{scope}`: Optional scope\n  - `{summary}`: Commit summary\n  - `{body}`: Detailed commit body\n  - `{footer}`: Optional footer\n  - `{ticket}`: Ticket ID\n  - `{ticketSeparator}`: Separator inserted if a ticket is provided\n- **CI Integration:** Optionally run a CI command before committing.\n- **Auto Ticket Extraction:** Optionally extract a ticket ID from your branch name using a custom regex.\n- **Push Support:** Use the `--push` flag to automatically push the commit to the remote repository after creation.\n- **Automatic Commit Type Suggestion:** The tool analyzes staged changes and suggests an appropriate commit type based on modified files.\n- **Signed Commits:** Use the `--sign` flag to create GPG-signed commits.\n- **Commit Statistics:** The `sc stats` command shows commit statistics for the last week.\n- **Local Configuration:** Global settings are stored in `~/.smart-commit-config.json`. To override these settings for a specific project, create a file named `.smartcommitrc.json` in your project directory. The local configuration will override the global configuration for that project.\n- **Interactive Setup:** Run `sc setup` to configure your preferences interactively.\n\n## Installation\n\n1. **Clone the repository and build:**\n\n   ```bash\n   git clone <repository_url>\n   cd smart-commit\n   npm install\n   npm run build\n   ```\n\n2. **Link globally:**\n\n   Ensure your `package.json` includes the following in the `bin` field:\n\n   ```json\n   \"bin\": {\n     \"smart-commit\": \"./dist/index.js\",\n     \"sc\": \"./dist/index.js\"\n   }\n   ```\n\n   Then run:\n\n   ```bash\n   npm link\n   ```\n\n   Now you can use `smart-commit` or `sc` in your terminal.\n\n## Usage\n\n### Creating a Commit\n\nRun the command:\n\n```bash\nsc commit [--push] [--sign]\n```\n\nYou will be prompted for:\n- **Commit Type:** Choose from the list. The tool may automatically suggest a type based on staged changes.\n- **Scope:** *(If enabled)* Enter an optional scope.\n- **Summary:** A short commit summary (auto-generated by default but editable).\n- **Body:** *(If enabled)* Your default editor will open for a detailed message.\n- **Footer:** *(If enabled)* Additional commit footer information.\n- **Ticket:** *(If enabled)*, which can be auto-extracted from the branch name if configured.\n- **Run CI tests:** *(If enabled)* Run the configured CI command before committing.\n- **Stage all changes:** *(Auto-add)* Stage all changes before creating the commit.\n- **Confirm:** Confirm and execute the commit.\n\nAdditional flags:\n- `--push`: After the commit, automatically push it to the remote repository.\n- `--sign`: Create a GPG-signed commit.\n\n### Running CI Tests Manually\n\n```bash\nsc ci\n```\n\n### Viewing Commit Statistics\n\n```bash\nsc stats\n```\n\nThis command displays commit statistics (using Git shortlog) for the last week.\n\n### Configuration\n\nView or update settings with:\n\n```bash\nsc config\n```\n\nFor example, to enable the ticket prompt and set a regex for ticket extraction:\n\n```bash\nsc config --enable-ticket true --ticket-regex \"^(DEV-\\\\d+)\"\n```\n\n### Interactive Setup\n\nRun the interactive setup wizard to configure your preferences:\n\n```bash\nsc setup\n```\n\nThis wizard will prompt you for:\n- Enabling/disabling prompts (scope, body, footer, ticket, CI)\n- Setting the ticket extraction regex\n- Defining the commit message template\n- Enabling auto-add\n- Setting the CI command\n\n### Local Configuration\n\nTo use project-specific settings, create a file named `.smartcommitrc.json` in your project's root directory. This file will override the global configuration (`~/.smart-commit-config.json`) for that project.\n\n## Configuration File\n\nThe global configuration is stored in your home directory as `~/.smart-commit-config.json`.
